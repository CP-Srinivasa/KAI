# Sprint 12 Contract — Training Job Record und Post-Training Evaluation

> **Canonical reference** für Sprint-12 kontrollierter Tuning-/Trainingsschritt,
> Training Job Manifest, Post-Training Evaluation Contract und Promotion Continuity.
>
> Upstream contracts: `docs/contracts.md §23`, invariants I-63–I-69.
> Upstream Sprint-11: `docs/sprint11_distillation_contract.md`, I-58–I-62.
> Upstream Sprint-8/9: `docs/tuning_promotion_contract.md`, I-40–I-50.

---

## Purpose

Sprint 12 schließt die Lücke zwischen `TuningArtifact` (Sprint 8) und `PromotionRecord`
(Sprint 8/9): Es gibt bisher **kein strukturiertes Artefakt**, das dokumentiert, dass
Training tatsächlich stattgefunden hat, mit welchen Parametern und mit welchem Ergebnis.

Sprint 12 fügt drei Artefakttypen hinzu:

| Artefakt | Rolle | Wann erzeugt |
|----------|-------|--------------|
| `TrainingJobRecord` | Manifest der Trainingsparameter (pre-training intent) | Vor dem Training — `prepare-training-job` CLI |
| `PostTrainingEvaluationSpec` | Verknüpfung von Job + Post-Training Evaluation | Nach Training + Evaluation — `link-training-evaluation` CLI |
| Extension: `PromotionRecord.training_job_record` | Optionaler Audit-Link im Promotion Record | Bei `record-promotion --training-job` |

---

## Core Separation — Nicht verhandelbar

| Konzept | Was es ist | Was es NICHT ist |
|---------|-----------|------------------|
| **Training Job Record** | Manifest: Wer trainiert mit was — kein Training | Kein Training-Job, kein API-Aufruf, kein Gewichtsupdate |
| **Post-Training Evaluation** | Evaluation des trainierten Modells gegen Teacher | Kein Gate-Bypass, keine automatische Promotion |
| **Promotion Gate** | `record-promotion` + G1–G6 wie in Sprint 8/9 | Unverändert — kein neuer Pfad |
| **Deployment** | Operator setzt `APP_LLM_PROVIDER` + Endpoint | Keine Platform-Code-Änderung |

**Vier explizite Trennungen — Sprint 12 erweitert Sprint 8:**

| Schritt | Sprint 8 | Sprint 12 Erweiterung |
|---------|----------|-----------------------|
| Tuning Manifest | `TuningArtifact` — Dataset + Model Base | `TrainingJobRecord` — explizit mit `target_model_id`, `job_id`, `row_count` |
| Training | Extern, Operator führt aus | Unverändert — Operator. Platform druckt nur `training_command` |
| Post-Training Evaluation | Manuell, ad-hoc | `PostTrainingEvaluationSpec` verknüpft Job + Eval-Report |
| Promotion | `record-promotion` (Sprint 8) | Optional: `--training-job` Flag für vollständige Audit-Kette |

## Datenmodelle

### `TrainingJobRecord`

```python
@dataclass
class TrainingJobRecord:
    """Immutable pre-training manifest. Records training intent only.

    Does NOT train a model. Does NOT call any external API.
    Generated by: research prepare-training-job

    Contract reference: docs/sprint12_training_job_contract.md §TrainingJobRecord
    Invariants: I-63, I-64, I-67
    """
    teacher_dataset: str          # absolute path to teacher JSONL (EXTERNAL_LLM only)
    model_base: str               # base model identifier (e.g. llama3.2:3b)
    target_model_id: str          # identifier for the resulting trained model
    training_format: str          # always "openai_chat"
    row_count: int                # teacher JSONL row count at manifest creation time
    job_id: str                   # uuid4 — unique per record
    tuning_artifact_path: str | None = None  # optional link to TuningArtifact
    notes: list[str] = field(default_factory=list)

    def to_json_dict(self) -> dict[str, object]:
        return {
            "record_type": "training_job",
            "generated_at": datetime.now(UTC).isoformat(),
            "job_id": self.job_id,
            "teacher_dataset": self.teacher_dataset,
            "model_base": self.model_base,
            "target_model_id": self.target_model_id,
            "training_format": self.training_format,
            "row_count": self.row_count,
            "tuning_artifact_path": self.tuning_artifact_path,
            "status": "pending",
            "notes": list(self.notes),
        }
```

**`status` bleibt immer `"pending"` bei Erstellung.** Das Training läuft extern. Kein
Platform-Code ändert dieses Feld nachträglich — ein neues Artefakt entsteht nach Training
(via `PostTrainingEvaluationSpec`).

**`job_id`** ist ein `uuid4`-String — wird von `save_training_job_record()` generiert,
falls nicht übergeben.

### `PostTrainingEvaluationSpec`

```python
@dataclass
class PostTrainingEvaluationSpec:
    """Links a training job record to its post-training evaluation report.

    Created after training is complete and evaluation has been run.
    Does NOT run evaluation. Does NOT trigger promotion.

    Contract reference: docs/sprint12_training_job_contract.md §PostTrainingEvaluationSpec
    Invariants: I-65, I-68
    """
    training_job_path: str         # absolute path to training_job_record.json
    trained_model_id: str          # model ID after training
    trained_model_endpoint: str    # endpoint URL of the trained model
    eval_report_path: str | None = None  # absolute path to evaluation_report.json
    notes: list[str] = field(default_factory=list)

    def to_json_dict(self) -> dict[str, object]:
        return {
            "record_type": "post_training_eval",
            "generated_at": datetime.now(UTC).isoformat(),
            "training_job_path": self.training_job_path,
            "trained_model_id": self.trained_model_id,
            "trained_model_endpoint": self.trained_model_endpoint,
            "eval_report_path": self.eval_report_path,
            "notes": list(self.notes),
        }
```

**`eval_report_path`** ist optional bei Erstellung (Operator kann zuerst den Spec anlegen,
dann Evaluation ausführen und Pfad manuell ergänzen). Wenn angegeben, muss die Datei existieren.

## Funktionen — `app/research/training.py`

### `save_training_job_record()`

```python
def save_training_job_record(
    output_path: Path | str,
    *,
    teacher_dataset: Path | str,
    model_base: str,
    target_model_id: str,
    row_count: int,
    tuning_artifact_path: Path | str | None = None,
    notes: list[str] | None = None,
) -> Path:
    """Write an immutable training job manifest.

    Does NOT train a model. Does NOT call any external API.
    Validates: teacher_dataset exists, row_count >= 1.

    Contract reference: docs/sprint12_training_job_contract.md §save_training_job_record
    Invariants: I-63, I-67
    """
```

**Validierungen:**
- `teacher_dataset` muss existieren → `FileNotFoundError` wenn nicht
- `row_count >= 1` → `ValueError` bei 0 oder negativ
- `target_model_id.strip()` muss nicht leer sein → `ValueError` wenn leer
- `tuning_artifact_path` muss existieren wenn angegeben → `FileNotFoundError` wenn nicht
- Generiert `job_id = str(uuid4())`
- `training_format` immer `"openai_chat"`

## Minimal Training Artifact Chain

Die vollständige Artefakt-Kette für einen kontrollierten Training-Run:

```
1. dataset-export --teacher-only > teacher.jsonl
        ↓ (optional)
2. prepare-tuning-artifact teacher.jsonl llama3.2:3b
        ↓ tuning_manifest.json
3. prepare-training-job teacher.jsonl llama3.2:3b kai-analyst-v1
        ↓ training_job_record.json
4. [EXTERN: Operator führt Training durch]
        ↓ trainiertes Modell verfügbar unter Endpoint
5. evaluate-datasets teacher.jsonl candidate.jsonl --save-report post_train_eval.json
        ↓ post_train_eval.json
6. link-training-evaluation training_job_record.json post_train_eval.json kai-analyst-v1 http://localhost:11434
        ↓ post_training_eval_spec.json
7. check-promotion post_train_eval.json
        ↓ (Exit 0 wenn G1–G6 bestanden)
8. record-promotion post_train_eval.json kai-analyst-v1 \
       --endpoint http://localhost:11434 \
       --training-job training_job_record.json \
       --operator-note "Training mit 450 Rows, alle Gates bestanden"
        ↓ promotion_record.json
9. [OPERATOR: APP_LLM_PROVIDER=companion + companion_model_endpoint=... setzen]
```

**Schritte 4 und 9 sind immer Operator-Aktionen.** Kein Platform-Code führt sie aus.

## Shadow Schema Canonicalization — Task 12.3

**Bekannte Inkonsistenz (seit Sprint 10/11):**
- `shadow.py` batch-Format: `divergence.priority_diff`, `divergence.relevance_diff`, `divergence.impact_diff`
- `evaluation.py` live-Format: `deviations.priority_delta`, `deviations.relevance_delta`, `deviations.impact_delta`

**Sprint-12 Resolution (I-69):**

Kanonisches Format: `deviations.*_delta` (wie `evaluation.py`).

Codex-Task 12.3:
- `shadow.py` `ShadowRunRecord` und `compute_divergence()` → schreiben `"deviations"` Feld (statt `"divergence"`)
- `DivergenceSummary` Felder bleiben intern gleich (`priority_diff`, etc.) — nur das JSONL-Key ändert sich
- Backward-Compat: `write_shadow_record()` schreibt BEIDE Keys (`"divergence"` als deprecated Alias, `"deviations"` als kanonisch) bis Migration abgeschlossen
- `compute_shadow_coverage()` in `distillation.py` bleibt kompatibel (normalisiert bereits beide Formate)
- Tests in `test_shadow.py` updaten: neues `"deviations"` Key prüfen

- `save_training_job_record()` validiert `teacher_dataset` existiert — keine Phantom-Records
- `save_post_training_eval_spec()` validiert `training_job_path` existiert — kein Dangling Link
- Kein Operator-Secret in einem Artefakt gespeichert
- Keine externen API-Calls in allen Sprint-12 Code-Pfaden
- `target_model_id` und `trained_model_endpoint` werden als Strings gespeichert — kein Shell-Aufruf
- Training-Command wird nur als String gedruckt — kein `subprocess.run()` oder `os.system()`
- Alle Artefakte sind write-once (I-38 gilt auch für Sprint-12 Artefakte)

## Invariant Summary (I-63 bis I-69)

Volltext in `docs/contracts.md §23`.

| ID | Regel |
|----|-------|
| I-63 | `TrainingJobRecord` ist ein platform-seitiges Pre-Training-Manifest. Kein Platform-Code führt Training aus, ruft Fine-Tuning APIs auf, oder modifiziert Modellgewichte. Training ist ausschließlich ein externer Operator-Prozess. |
| I-64 | Ein `TrainingJobRecord` mit `status="pending"` repräsentiert kein trainiertes Modell. Der Operator muss Training extern durchführen, bevor Post-Training Evaluation beginnen kann. |
| I-65 | Post-Training Evaluation MUSS dieselben Gates G1–G6 bestehen wie Pre-Promotion Evaluation. `validate_promotion()` ist der kanonische Gate-Check — kein Sprint-12 Bypass erlaubt. |
| I-66 | Ein trainiertes Modell ist erst aktiv, wenn der Operator `APP_LLM_PROVIDER` und `companion_model_endpoint` rekonfiguriert. Kein Sprint-12 Code ändert Routing (I-42 gilt). |
| I-67 | Das Teacher-Dataset in `TrainingJobRecord` MUSS ausschließlich `analysis_source=EXTERNAL_LLM` Rows enthalten. `INTERNAL`, `RULE` und Shadow-Records DÜRFEN NICHT als Training-Input verwendet werden (I-16, I-19, I-53 gelten). |
| I-68 | `record-promotion` bleibt der einzige Promotion-Gate. `TrainingJobRecord` und `PostTrainingEvaluationSpec` sind Audit-Artefakte — sie lösen keine Promotion aus und ersetzen den Gate-Check nicht. |
| I-69 | Sprint-12 kanonisiert das Shadow-JSONL-Schema: `shadow.py` schreibt `"deviations"` (mit `priority_delta`, `relevance_delta`, `impact_delta`) als kanonisches Key — übereinstimmend mit `evaluation.py`. `"divergence"` bleibt als deprecated Alias bis alte Shadow-Files migriert sind. `compute_shadow_coverage()` normalisiert weiterhin beide Formate. |
