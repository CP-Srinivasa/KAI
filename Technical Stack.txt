16. Multi-Agent Collaboration Rules

This repository is intended to be worked on by multiple coding assistants and agents.

To preserve consistency:

all agents must follow the same architecture,

all agents must avoid duplicative module creation,

all agents must report changed files and assumptions,

all agents must preserve tests,

all agents must keep unresolved source cases explicit.

17. Success Criteria

The project is successful when it provides:

a stable ingest-and-analyze foundation,

high-quality filtered outputs,

actionable alerts,

research-ready summaries,

signal-ready but not recklessly auto-executed outputs,

clean extensibility for future trading integrations.

18. Final Constraint

This is not a toy project.
It should be implemented as a clean, extensible, production-oriented foundation that can grow over time without becoming fragile or chaotic.
